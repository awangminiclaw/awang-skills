# Higgs-Pro: 好萊塢得獎大片分鏡提示詞系統

這是 Higgs skill 的進階版本，專門設計用於生成 AI 影片的分鏡提示詞。核心特色是根據秒數自動推導分鏡規則，並輸出符合「AI 影片首尾幀概念」的提示詞格式。

---

## 核心概念：電影片段而非完整故事

### 🎬 正確理解：電影預告的「切片」

**首尾幀控制影片 ≠ 完整故事**
**首尾幀控制影片 = 從電影中切出的一小段**

#### 核心問題：「在這個故事世界中，哪個瞬間最震撼？」

這個瞬間可以是：
- 🎇 動作瞬間：特技演員完成不可能任務、賽車衝線、棒球全壘打
- 💘 情感瞬間：情人街頭遇見、告別擁抱、重聚時刻
- 💥 災難瞬間：爆炸、崩塌、海嘯、地震
- 🦸 變身瞬間：超人變身、力量覺醒、超能力爆發
- 🎭 戲劇瞬間：關鍵對決、真相揭露、勝利時刻

#### 範例理解

**主題**：不可能的任務
- **完整的電影故事**：間諜拯救世界、諜對諜、科技裝備...（2小時）
- **我們呈現的10秒**：大樓爆炸崩塌的瞬間
  - 首幀：大樓開始崩塌
  - 動態：碎片飛散、鋼筋斷裂
  - 尾幀：大樓完全collapse

**主題**：愛在日落巴黎時
- **完整的電影故事**：愛情、分離、重逢...（2小時）
- **我們呈現的10秒**：情人街頭相遇的瞬間
  - 首幀：兩人目光交會
  - 動態：慢動作走向對方、背景虛化
  - 尾幀：擁抱在一起的特寫

**主題**：灌籃高手
- **完整的故事**：高中籃球、友情、成長...（整個系列）
- **我們呈現的10秒**：最後一投的瞬間
  - 首幀：球員起跳
  - 動態：空中旋轉、時間凝結
  - 尾幀：球進籃框的瞬間

張力等級變化必須平滑：

```
正確: 2 → 5 → 9 → 4 (平滑過渡)
錯誤: 2 → 9 → 2 → 9 (跳動太大)
```

### 角色一致性強制協議

在所有幀中使用 `[Image 1]`、`[Image 2]` 等標記：

```
【幀1】• Character: [Image 1] - Senior combat surgeon, battle-worn...
【幀2】• Character: [Image 1] - Same surgeon, CLOSE-UP on face...（同一角色！）
【幀3】• Character: [Image 2] - Apprentice surgeon, [Image 1]'s hands...（兩個角色！）
```

### 禁止事項

❌ 禁止在不同幀使用不同的渲染引擎  
❌ 禁止突然改變光照方向或色溫  
❌ 禁止角色服裝/外觀在不同幀突然改變  
❌ 禁止環境元素突然出現或消失  
❌ 禁止攝影機風格在不同幀突然改變

## 核心概念：AI 影片首尾幀輸出格式

### ⚡ 首尾幀的真正目的：視覺衝擊點（Impact Moments）

**首尾幀控制影片的核心不是「講故事」，而是創造視覺震撼點。**

#### 首帧（起始帧）= 衝擊畫面
設計「眼球一秒被抓走」的瞬間：
- 🏙️ 城市突然核爆
- 🦸 人突然變身超人
- 💥 正常街景突然爆炸
- 🌊 海嘯瞬間吞沒城市
- ⚡ 閃電劈開天空

#### 尾帧（终点帧）= 戲劇性終點
設計「讓人屏住呼吸」的結果畫面：
- 🏙️ 核爆後的狼藉城市
- 🦸 變身完成後的帥氣姿勢
- 💥 爆炸後的煙硝彌漫
- 🌊 海啸後的平靜海面
- ⚡ 閃電後的燃燒廢墟

#### 動態提示詞 = 過渡動作
描述首尾幀之間的**轉變過程**：
- 從正常到災難的崩解瞬間
- 從人到超人的變身動畫
- 從平靜到爆炸的衝擊波

### 輸出結構（3分鏡範例）

```
【圖像提示詞 1】第1幀 - 衝擊畫面（獨立輸出）
【圖像提示詞 2】第2幀 + 【動態提示詞 1-2】（過渡）
【圖像提示詞 3】第3幀 + 【動態提示詞 2-3】（過渡）
【圖像提示詞 4】第4幀 - 戲劇性終點（獨立輸出）+ 【動態提示詞 3-4】
```

### 主題選擇方向

**❌ 錯誤方向**（只有故事，沒有衝擊）：
- 外科醫生手術過程
- 街頭音樂家演奏
- 上班族走路回家

**✅ 正確方向**（有視覺衝擊點）：
- 外科醫生手術中，手術室突然被炸彈擊中
- 街頭音樂家演奏時，吉他突然變成火焰
- 上班族走路時，路面突然裂開

## 工作流程

### Phase 1: 主題選擇（接收主題）

**動態生成邏輯**：
1. **讀取資料庫**：使用 `view_file` 讀取主題資料庫
2. **混合主題**：
   - 從資料庫隨機選擇最多 7 個主題
   - 生成 8 個全新主題（多元類型：奇幻、科幻、恐怖、歷史、抽象）
3. **呈現選單**：顯示編號列表（1-15）+ 第 16 項「自行輸入」

**選單格式**：
```
1. [From DB] 賽博龐克夜市（Cyberpunk Street Market）
2. [New] 蒸汽龐克飛船（Steampunk Airship）
...
15. [New] 深淵巨獸（Abyssal Kaiju）
16. 自行輸入關鍵字
```

**停止並等待**：**不要**立刻顯示風格選單，等待使用者選擇。

> [!IMPORTANT]
> **資料庫更新規則**：
> 當使用者選擇主題（或輸入自訂），檢查是否存在於主題資料庫。如果是新主題，在最後「儲存到資料庫」階段追加。

### Phase 2: 風格選擇

主題確認後，顯示風格選單：

**選項 1：真人實拍（Photorealistic / Live Action）**
- 無 Prompt Tags
- 35mm film grain, natural lighting
- Arri Alexa, Kodak Vision3, Anamorphic Lens

**選項 2：Cinematic Hyper-Realism (CGI)**
- Love, Death & Robots / UE5 標準
- MetaHuman quality, Subsurface Scattering
- 必須在提示詞開頭注入：
> `Unreal Engine 5 render, Love Death & Robots animation style：1.2, MetaHuman quality, Subsurface Scattering (SSS) skin texture, ACES color system, Hollywood blockbuster composition, Cinemascope 2.35:1, volumetric lighting, anamorphic lens flares, depth of field, stylized realism, ultra-detailed organic textures, 8k resolution, 3D CGI masterpiece.`

**Negative Prompt**：
> `--no sci-fi, technology, cyborg, robot, mechanical parts, wires, metal skin, glitch, cyberpunk, photograph, camera noise.`

### Phase 3: 場景變化生成

基於主題和風格，生成 **15 個獨特場景概念**：
- 每個概念必須視覺化且適合高預算好萊塢製作
- 保持類型真實性（歷史題材不要加科幻元素）
- 編號列表（1-15）+ 標題 + 一句視覺摘要
- **不要**生成完整提示詞

### Phase 4: 圖像提示詞生成

使用者選擇後：
- **單選**：生成特定概念
- **多選/自訂**：融合所有選擇元素成一個完整故事/場景

**提示詞結構（高密度敘事塊）**：
```
Image Prompt:

**VISION CONCEPT**: A hyper-realistic cinematic masterpiece depicting [Subject] at [Location] involving [Action]. The visual style follows the aesthetic of 'Love, Death & Robots'...

**SUBJECT_DATA**:
• Character: [Details]
• MetaHuman Skin: [Details]
• Grooming: [Details]
• Attire: [Details]

**ENVIRONMENTAL_DATA**:
• Foreground: [Details]
• Midground: [Details]
• Atmosphere: [Details]

**CINEMATOGRAPHY & LIGHT**:
• Camera: Virtual Arri Alexa 65, Panavision C Series Anamorphic Lenses, f/2.0
• Lighting: [Details]
• Post-Process: Bleach Bypass, 35mm film grain, Chromatic aberration, Halation

**TECH_STACK**: Unreal Engine 5 (Lumen GI, Nanite Geometry, Ray-Traced Reflections)

[IF CGI STYLE]: (以下為虛構情節，所有角色皆為虛構或經官方開放創作使用)
```

### Phase 4: 秒數選單（新增功能）

> **⚠️ Kling 3.0 時長規則**：
> - 最短時長：3秒
> - 最長時長：15秒
> - 時長單位：整數秒（沒有半秒）
> - 可選時長：3秒、6秒、8秒、10秒、15秒
> - 每段分鏡最短：3秒

根據選擇的主題、風格、場景，顯示秒數選項：

```

請輸入 1-5 選擇秒數：
```

**停止並等待**：**不要**立刻顯示分鏡選項，等待使用者選擇秒數。

### Phase 5.5: 視覺線與分鏡規劃（輸出前必經步驟）

> **⚠️ 重要**：在生成任何圖像提示詞或動態提示詞之前，必須按照以下順序完成規劃並獲得使用者確認。跳過任何步驟都會導致邏輯錯誤、方向不對、幀與幀不連貫。

#### 工作流程順序（嚴格遵守）

```
第一步：視覺線（Visual Line）
   ↓ 畫出觀眾視線的軌跡，決定畫面的流動方向
   
第二步：攝影機位置/運鏡（Camera Setup）
   ↓ 根據視覺線決定鏡頭放在哪裡、如何移動
   
第三步：角色動作設計（Action Design）
   ↓ 根據視覺線和運鏡設計角色走位和動作
   
第四步：確認圖片（Frame Definition）
   ↓ 每一幀的具體狀態自然出現
   
第五步：動態提示詞（Motion Prompts）
   ↓ 描述從幀A到幀B的變化過程
```

#### 為什麼是「視覺線」而不是「故事線」

**故事線**：敘事角度的劇情發展
**視覺線**：視覺角度的觀眾視線軌跡

舉例：
- 故事線：「衝擊波摧毀街道」
- 視覺線：「觀眾視線從遠方衝擊波 → 跟隨衝擊波接近 → 看到街道被摧毀」

**視覺線決定**：
- 攝影機應該放在哪裡？
- 鏡頭應該如何移動？
- 角色應該如何走位？
- 觀眾應該看到什麼？

---

#### 第一步：視覺線（Visual Line）

畫出觀眾視線的軌跡：

```
【視覺線確認】
- 起點：觀眾第一眼看到什麼？
- 軌跡：視線如何移動？從哪裡到哪裡？
- 終點：視線最後停留在哪裡？
- 核心動作：什麼視覺元素驅動視線移動？
```

**視覺線範例（衝擊波場景）**：
```
視覺線：從遠方衝擊波 → 跟隨接近 → 看到街道被摧毀

視線軌跡：
- 起點：遠處的蘑菇雲/衝擊波前線
- 移動：衝擊波從遠方接近鏡頭
- 終點：衝擊波通過鏡頭，看到毀滅全景

視覺元素驅動：
- 衝擊波前線（可見的扭曲牆）帶動視線
- 飛向鏡頭的碎片加強視線方向感
```

**視覺線範例（對戰場景）**：
```
視覺線：從對峙 → 跟隨出拳 → 看到倒地

視線軌跡：
- 起點：兩人對峙，視線在兩人中間
- 移動：跟隨攻擊者的出拳動作
- 終點：防守者倒地，視線落在倒地者身上

視覺元素驅動：
- 攻擊者的拳頭帶動視線
- 防守者的身體移動跟隨視線
```

---

#### 第二步：攝影機位置/運鏡（Camera Setup）

根據視覺線決定鏡頭：

```
【攝影機確認】
- 鏡頭位置：在哪裡？高度？角度？
- 鏡頭動作：靜止？推拉？搖鏡？跟隨？
- 鏡頭目的：為什麼這樣放？服務視覺線的哪一段？
```

**攝影機範例（衝擊波場景）**：
```
視覺線：衝擊波從遠方接近 → 鏡頭位置：街道水平，面向遠方
                           → 鏡頭動作：靜止，讓衝擊波接近
                           
視覺線：毀滅全景 → 鏡頭位置：略微後拉
               → 鏡頭動作：緩慢後拉，揭示全景
```

**攝影機範例（對戰場景）**：
```
視覺線：對峙 → 鏡頭位置：中景，涵蓋兩人
             → 鏡頭動作：靜止，強調對峙張力

視覺線：出拳 → 鏡頭位置：跟隨出拳者
             → 鏡頭動作：推進到中近景，強調出拳瞬間

視覺線：倒地 → 鏡頭位置：後拉，涵蓋倒地過程
             → 鏡頭動作：緩慢後拉，展示完整倒地
```

---

#### 第三步：角色動作設計（Action Design）

根據視覺線和運鏡設計動作：

```
【動作設計確認】
- 動作軌跡：角色從哪裡移動到哪裡？
- 力量方向：力量從哪裡發出？往哪裡傳遞？
- 動作結果：動作完成後的狀態是什麼？
```

**動作範例（衝擊波場景）**：
```
視覺線：衝擊波接近 → 動作：物體被推向鏡頭
- 力量來源：遠方核爆
- 方向：從核爆點 → 向外擴散 → 推向鏡頭

視覺線：衝擊波通過 → 動作：物體落下
- 力量：重力
- 方向：從空中 → 落向地面
```

**動作範例（對戰場景）**：
```
視覺線：出拳 → 動作：攻擊者出拳
- 力量來源：攻擊者身體扭轉
- 軌跡：肩膀 → 手臂延伸 → 拳頭 → 防守者

視覺線：倒地 → 動作：防守者向後倒
- 力量：被擊中的衝擊力
- 軌跡：防守者向後傾斜 → 超過平衡點 → 倒地
```

#### 第四步：確認圖片（Frame Definition）

根據視覺線、運鏡、動作，每一幀的狀態自然確定：

```
【幀狀態確認表】

| 元素 | 幀1（起點） | 幀2（轉折） | 幀3（終點） |
|------|------------|-------------|------------|
| 視覺線位置 | 視線在哪 | 視線移動到哪 | 視線停在哪 |
| 核心元素 | 狀態 | 狀態 | 狀態 |
| 鏡頭 | 位置/動作 | 位置/動作 | 位置/動作 |
| 張力 | 1-10 | 1-10 | 1-10 |
```

---

#### 第五步：動態提示詞（Motion Prompts）

只有在確認以上四步後，才能寫動態提示詞：

```
【動態提示詞格式】

【動態 X-Y】從幀X到幀Y
**視覺線移動**：視線從 [位置A] 移動到 [位置B]
**動作變化**：[具體動作描述，包含方向]
**運鏡**：[鏡頭如何移動]
**環境變化**：[環境如何變化]
**時長**：X秒
```

#### 錯誤順序 vs 正確順序

| 錯誤順序 | 問題 |
|---------|------|
| 跳過視覺線 → 直接寫運鏡 | 不知道鏡頭要跟隨什麼 |
| 跳過運鏡 → 直接寫動作 | 不知道動作要在哪裡發生 |
| 跳過動作 → 直接寫圖片 | 不知道角色在做什麼 |
| 跳過圖片確認 → 直接寫動態 | 動態描述與幀狀態不匹配 |

#### 正確順序的優勢

```
視覺線 → 決定觀眾看什麼
   ↓
攝影機 → 決定如何呈現視覺線
   ↓
動作 → 決定角色如何在鏡頭前移動
   ↓
圖片 → 每一幀自然確定
   ↓
動態 → 變化過程清晰可寫
```

#### 敘事規劃輸出格式

在輸出任何提示詞之前，先呈現以下規劃：

```
=== 敘事規劃確認 ===

時間線：0秒 ────────────── 時間點A ────────────── 時間點B

       【幀1】              【幀2】              【幀3】（如有）
       狀態描寫            狀態描寫            狀態描寫
       ↓                    ↓                   ↓
      關鍵元素狀態        關鍵元素狀態        關鍵元素狀態
      張力等級：X          張力等級：Y          張力等級：Z

       ─────動態1-2─────       ─────動態2-3─────
       變化描述              變化描述
       方向確認              方向確認
```

#### 幀狀態定義（必填）

對於每一幀，必須定義：

| 元素 | 幀1（首幀） | 幀2（中幀/尾幀） | 幀3（如有） |
|------|------------|-----------------|------------|
| 關鍵元素A | 狀態 | 狀態 | 狀態 |
| 關鍵元素B | 狀態 | 狀態 | 狀態 |
| 關鍵元素C | 狀態 | 狀態 | 狀態 |
| 張力等級 | 1-10 | 1-10 | 1-10 |

#### 動態變化定義（必填）

對於每一段動態，必須定義：

| 元素 | 動態1-2（幀1→幀2） | 動態2-3（如有） |
|------|-------------------|-----------------|
| 元素A | 從 狀態X → 狀態Y | 從 狀態Y → 狀態Z |
| 元素B | 變化描述 | 變化描述 |
| 方向 | 明確方向 | 明確方向 |

#### 物理方向確認表（必填）

| 元素 | 幀1 → 幀2 方向 | 幀2 → 幀3 方向 |
|------|----------------|----------------|
| 主要物體 | 動作方向 | 動作方向 |
| 衝擊力 | 來源 → 目標 | 來源 → 目標 |
| 碎片/粒子 | 飛行方向 | 飛行/落下方向 |

#### 衝擊波/爆炸場景特別確認

對於衝擊波/爆炸場景，必須明確：

```
【衝擊波確認】
- 幀1衝擊波狀態：不存在 / 存在於位置X
- 幀2衝擊波狀態：到達鏡頭 / 在位置Y
- 幀3衝擊波狀態：已通過 / 在位置Z
- 方向：從爆炸點 → 向外擴散 → 接近/遠離鏡頭

【物體運動確認】
- 幀1物體狀態：靜止 / 運動中
- 幀2物體狀態：被推向鏡頭 / 被推離鏡頭 / 靜止
- 幀3物體狀態：落下 / 散落 / 靜止
```

#### 格鬥場景特別確認

對於格鬥場景，必須明確：

```
【動作確認】
- 幀1動作狀態：對峙 / 準備攻擊
- 幀2動作狀態：出拳瞬間 / 被擊中瞬間
- 幀3動作狀態：倒地 / 結束姿勢

【力量方向確認】
- 攻擊方向：從攻擊者 → 延伸出去
- 衝擊力：從擊中點 → 向對手後方傳遞
- 倒地方向：向後 / 向側 / 向前
```

**確認流程**：

1. 輸出上述敘事規劃表格
2. **停止並等待使用者確認**
3. 使用者確認後，才輸出具體的圖像提示詞和動態提示詞

**錯誤案例**：
- ❌ 幀1已有衝擊波，但應該是平靜狀態
- ❌ 幀2沒有物體飛向鏡頭，但動態描述說物體飛向鏡頭
- ❌ 幀之間的物理狀態不連貫

**正確案例**：
- ✅ 幀1平靜無衝擊波，幀2衝擊波到達，動態描述衝擊波「出現並接近」
- ✅ 幀2車開始飛向鏡頭，幀3車殘骸散落，動態描述「飛過鏡頭後落下」

### Phase 6: 分鏡設計輸出（AI 影片首尾幀格式）

根據選擇的分鏡數量 N，輸出：
- **N+1 張圖像提示詞**
- **N 條動態提示詞**

> **⚠️ 風格一致性強制要求**：
> 在生成多幀提示詞時，必須確保所有幀使用相同的核心風格參數：
> - 相同的渲染引擎標籤（逐字複製到每一幀）
> - 相同的攝影機設定
> - 相同的色彩系統
> - 一致的光照方向
> - 一致的環境粒子
> 
> **禁止修改核心風格區塊！每一幀必須逐字複製相同的 TECH_STACK、RENDER STYLE、CAMERA、POST-PROCESS。**

> **⚠️ 動態提示詞物理細節要求**：
> 動態提示詞必須包含具體的動作細節，不能籠統描述：
> - **具體招數**：什麼拳（直拳、勾拳、上勾拳）、什麼踢（側踢、迴旋踢、前踢）、什麼防守（格擋、閃避、後退）
> - **身體力學**：力量如何傳遞、身體如何扭轉、重心如何轉移、腳步如何移動
> - **對手反應**：對手如何防禦、如何被衝擊、如何失去平衡、如何跌倒
> - **環境物理**：水花如何飛起、塵土如何飛揚、物體如何被影響
> - **時間感**：快動作還是慢動作、衝擊瞬間是否有停頓
> - **⚠️ 物理方向（關鍵）**：所有運動必須有明確方向
>   - 衝擊波：從爆炸點「向外擴散」，物體被「推開」（飛向相機方向）
>   - 爆炸：碎片「向外飛」，不是「往內收縮」
>   - 推/拉：明確「toward camera」或「away from camera」
>   - 重力：物體下落方向
>   - 風：明確風從哪裡來、往哪裡去
> 
> **災難場景物理方向範例**：
> - **核爆衝擊波**：爆炸在遠方 → 衝擊波從遠方「接近相機」→ 物體被推開「飛向相機」
>   - 關鍵字：expanding outward FROM explosion, approaching camera, debris flies TOWARD camera, pushed away FROM explosion point
> - **海嘯**：海浪從遠方「接近海岸」→ 物體被推向「內陸方向」
> - **龍捲風**：風「旋轉吸入」物體 → 物體「向上螺旋」
> 
> **格鬥場景物理方向範例**：
> - **出拳**：拳頭從攻擊者「延伸出去」→ 對手被「推向後方」
>   - 關鍵字：fist extends outward, opponent pushed backward, force transfers through impact point
> - **踢腿**：腿從攻擊者「向外踢出」→ 對手被「踢飛」
> - **跌倒**：身體「向後傾斜」→ 重心「超過平衡點」→ 身體「向後倒下」
> 
> **錯誤範例**：「Fighter A punches Fighter B, B falls down」
> **正確範例**：「Fighter A throws a right hook（右勾拳）with full body rotation, left foot pivoting, right arm extending in an arc toward Fighter B's jaw. Fighter B raises left arm in cross-block, absorbing the impact. The force pushes B's shoulder backward, left foot loses grip on wet stone, body pivots on right foot, center of gravity shifts past the tipping point. B attempts to regain balance with left arm windmilling, but momentum carries body backward. B's back hits the wet cobblestones with a splash, arms instinctively spread to break fall.」
> 
> **衝擊波錯誤範例**：「Shockwave reaches camera, cars flip inward, debris flies inward」
> **衝擊波正確範例**：「Shockwave expands outward from distant explosion, approaches camera position. Cars are lifted and thrown FORWARD toward camera, AWAY from explosion. Debris flies TOWARD camera. Building facades blow OUTWARD, shards spray TOWARD lens. Everything is pushed AWAY from explosion point, TOWARD camera.」

#### 強制輸出格式範本

每個圖像提示詞**必須**包含以下核心區塊（所有幀完全相同）：

```
**TECH_STACK**: Unreal Engine 5 (Lumen GI, Nanite Geometry, Ray-Traced Reflections), Movie Render Queue (64 spatial samples)

**RENDER STYLE**: Love Death & Robots animation style：1.2, MetaHuman quality, Subsurface Scattering (SSS) skin texture, ACES color system, Hollywood blockbuster composition, Cinemascope 2.35:1, volumetric lighting, anamorphic lens flares, depth of field, stylized realism, ultra-detailed organic textures, 8k resolution, 3D CGI masterpiece.

**CAMERA**: Virtual Arri Alexa 65, Panavision C Series Anamorphic Lenses, f/2.0

**POST-PROCESS**: Bleach Bypass process (High contrast, desaturated), Physical 35mm film grain, Chromatic aberration, Halation
```

## 鏡頭運動資料庫（Camera Movement Library）

### 基礎運鏡（Dolly, Pan, Tilt, Truck）
- **Dolly In**: 鏡頭緩慢推向主體，電影級光照，淺景深
- **Dolly Out**: 鏡頭緩慢拉離主體，揭示更多環境
- **Fast Dolly In (Rush)**: 鏡頭快速衝向主體，創造緊迫感
- **Pan Left/Right**: 鏡頭水平左右移動
- **Tilt Up/Down**: 鏡頭垂直上下移動
- **Truck Left/Right**: 鏡頭左右平移，強烈視差
- **Pedestal Up**: 鏡頭從腰部高度上升到眼睛高度

### 軌道與吊臂運鏡（Orbital & Crane）
- **Orbit 180/360**: 鏡頭環繞主體180度或360度
- **Slow Cinematic Arc**: 鏡頭以寬闊弧線緩慢移動
- **Crane Up/Down**: 吊臂上升或下降
- **Crane Overhead**: 吊臂從上方越過主體

### 變焦與鏡頭效果（Zoom & Lens）
- **Smooth Zoom In/Out**: 鏡頭靜止，緩慢變焦
- **Crash Zoom In/Out**: 突然快速變焦
- **Rack Focus**: 焦點從主體轉移到背景
- **Reveal From Blur**: 從模糊漸變清晰

### 空拍與無人機（Aerial & Drone）
- **Drone Flyover**: 高空空拍前進
- **Epic Drone Reveal**: 無人機上升揭示全景
- **FPV Drone**: 第一人稱視角高速穿梭

### 主體追蹤（Subject Tracking）
- **Leading Shot**: 主體走向鏡頭，鏡頭同步後退
- **Following Shot**: 主體背對鏡頭，鏡頭跟隨
- **Side Tracking**: 側面跟拍，背景動態模糊
- **POV Walk**: 第一人稱行走視角

### 創意與高級運鏡（Creative & Advanced）
- **Vertigo Effect (Dolly Zoom)**: 希區考克式眩暈效果
- **Through Shot**: 穿越前景物體
- **Reveal From Behind**: 從障礙物後方揭示
- **Dutch Angle**: 荷蘭角，傾斜構圖
- **Whip Pan**: 快速橫搖轉場
- **Handheld Documentary**: 自然手持晃動
- **MO-CAP / ROBOTIC ARM**: 機械臂運鏡，零晃動

## 分鏡節奏設計準則

### Kling 3.0 時長限制

> **⚠️ 重要規則**：
> - 分鏡時長最短單位：**3秒**
> - 總時長範圍：**3秒 - 15秒**
> - 每段分鏡必須 ≥ 3秒
> - 例如：10秒影片可分配為 3-3-4秒（3分鏡）或 3-3-2-2秒（4分鏡，需注意最後兩段）
> - 例如：15秒影片可分配為 3-3-3-3-3秒（5分鏡）或 4-4-4-3秒（4分鏡）

### 好萊塢得獎大片分鏡結構

**3秒結構（1分鏡）**：
- 單一連續動作，一鏡到底
- 適合：單一衝擊瞬間、出拳、爆炸、變身

**6秒結構（2分鏡）**：
- 分鏡1：對峙/建立 → 分鏡2：衝擊/結束
- 每段約3秒
- 適合：簡單對比、單一動作完成

**8秒結構（2-3分鏡）**：
- 分鏡1：建立 → 分鏡2：衝突/轉折 → 分鏡3：解決（如選3分鏡）
- 每段約 3-4 秒（2分鏡）或 3-3-2 秒（3分鏡，注意最後一段）
- 適合：簡單敘事、產品展示

**10秒結構（3-4分鏡）**：
- 分鏡1：開場
- 分鏡2：發展
- 分鏡3：高潮
- 分鏡4：結尾 - 每段約 3.3 秒
- 4分鏡需調整為 3-3-2-2 秒分配
- 適合：完整小故事、電影預告片段

**15秒結構（4-5分鏡）**：
- ACT 1: 開場（1分鏡，約3秒）
- ACT 2: 發展（1-2分鏡，約3-6秒）
- ACT 3: 高潮（1分鏡，約3秒）
- ACT 4: 結尾（1分鏡，約3秒）
- 適合：品牌故事、紀錄片片段

## 張力等級設計

每個分鏡段應指定張力等級（1-10）：

| 等級 | 描述 | 適用時機 |
|------|------|---------|
| 1-3 | 低張力 | 建立鏡頭、氛圍鋪陳 |
| 4-7 | 中張力 | 衝突發展、情緒堆疊 |
| 8-10 | 高張力 | 高潮時刻、關鍵轉折 |

## 好萊塢得獎大片參考

### 最佳實踐（Best Picture Winners）

**視覺敘事原則**：
1. **首幀必須吸睛**：使用強烈視覺元素或動作
2. **終幀必須有力**：情緒釋放或視覺高潮
3. **中間幀串聯敘事**：每個過渡都有意義
4. **運鏡服務故事**：不是為了炫技而運鏡

**燈光與色彩**：
- 冷暖對比（Cool/Warm Contrast）
- 主光源方向一致性
- 情緒曲線：暗→亮→暗 或 亮→暗→亮

**聲音設計**：
- 環境音（Ambience）
- 效果音（SFX）
- 對話（可選）
- 音樂節奏
